## 図 3: search-api プロキシ化のビフォー / アフター

```text
[Phase 3]
User --HTTP--> Cloud Run search-api
                |
                |- lifespan: load E5Encoder (~1.1GB)
                |- lifespan: load lgb.Booster (GCS から model.txt)
                |
                |- encoder.encode_queries(query)    <-- 自前で推論
                |- Meili + BQ VECTOR_SEARCH -> RRF
                |- booster.predict(features)        <-- 自前で推論
                |
                v
             top-K response

[Phase 4]
User --HTTP--> Cloud Run search-api (2Gi)
                |
                |- lifespan: VertexEndpointEncoder DI
                |- lifespan: VertexEndpointReranker DI
                |
                |- encoder_client.embed(query, "query")  --HTTPS--> encoder Endpoint
                |- Meili + BQ VECTOR_SEARCH -> RRF
                |- reranker_client.predict(features)     --HTTPS--> reranker Endpoint
                |
                v
             top-K response
```

## 図 6: CPR コンテナと Model Registry のライフサイクル

```text
 ローカル開発
    |
    v
 コード変更 (jobs/containers/reranker/ 配下 or entrypoints/reranker_server.py)
    |
    v
 GitHub Actions deploy-reranker-image.yml
    |  ビルド + AR push
    v
 Artifact Registry: property-reranker:<sha>

 train_pipeline 実行
    |
    v
 train_reranker コンポーネント
    |  rank_trainer.train -> model.txt + metrics
    |  Vertex Experiments に log_params / log_metrics (W&B と dual)
    |  BQ mlops.training_runs に INSERT (従来通り)
    v
 evaluate コンポーネント  (NDCG@10 >= threshold?)
    |  pass
    v
 register_reranker コンポーネント
    |  ModelUploadOp (serving_container = property-reranker:<sha>,
    |                 artifact_uri = gs://.../lgbm/<date>/<run_id>/)
    |  -> Model Registry: property-reranker v<N> (alias=staging)
    |  Model.deploy(endpoint=property-reranker-endpoint,
    |               traffic_split={v<N>: 10, v<N-1>: 90})
    v
 オンライン観測 (NDCG、latency、drift)
    |
    v
 scripts/ops/promote.py reranker v<N>
    |  Registry alias production <- v<N>
    |  Endpoint traffic_split = {v<N>: 100}
    |  Endpoint.undeploy_model(v<N-1>)
    v
 本番トラフィック全量 = v<N>
```

## 図 7: 6 ファイル parity invariant

```text

      |
      | 10 列: rent, walk_min, age_years, area_m2, ctr, fav_rate,
      |        inquiry_rate, me5_score, lexical_rank, semantic_rank

      |         ranking_log.features RECORD schema (10 列 FLOAT64 NULLABLE)
      |
      +--(4)-- monitoring/validate_feature_skew.sql
      |         property-side 7 列を UNPIVOT で列挙
      |
      +--(5)-- tests/parity/test_feature_parity_sql_ranker.py
      |         (1)〜(4) の lockstep を AST で検査
      |
      +--(6)-- infra/modules/vertex/main.tf           <Phase 4 新規>
```
