# 構成タイムライン

`Cloud Run + BigQuery` 中心の GCP MLOps 教材として再構成した場合の進行メモ。

---

## 本編タイムライン

| No | セクション | 想定尺 | 形式 | 要点 |
|---|---|---:|---|---|
| 1 | タイトル / 前提 | 0:35 | スライド | ML 基礎は既知前提 |
| 2 | MLOps パイプライン全体像 | 1:00 | スライド | データ加工 -> 学習 -> 推論 -> 再学習 |
| 3 | GCP 技術と工程の対応 | 1:00 | スライド | Dataform / BigQuery / Cloud Run / Pub/Sub |
| 4 | 推論工程の流れ | 1:10 | スライド | query embedding -> VECTOR_SEARCH -> rerank |
| 5 | ログ / フィードバック / 再学習 | 1:00 | スライド | ranking_log / feedback_events / training_runs |
| 6 | デモ | 4:00 | 画面収録 | `make ops-search` / `make ops-ranking` / `make ops-check-retrain` |
| 7 | まとめ | 0:30 | スライド | API とデータ基盤を一体で捉える |

**合計:** 約 9 分 15 秒

---

## 伝え方の軸

- 「検索 API の作り方」よりも「GCP 上での MLOps 構成」に寄せる
- `Cloud Run Service` と `Cloud Run Jobs` を並べて見せる
- `BigQuery` を保存先だけでなく候補抽出基盤として扱う
- 再学習の話は条件判定とイベント連携まで触れる
