# ナレーション台本

簡易版です。詳細版は `../02_ナレーション台本.md` を参照してください。

---

## [Slide 00]

> Cloud Run と BigQuery を主役にした GCP MLOps パイプラインの教材です。

## [Slide 01]

> ML 基礎は既知前提で、今回は MLOps 工程と GCP 技術の関係理解に集中します。

## [Slide 02]

> 題材は不動産検索ですが、学びたいのは軽量 MLOps の工程設計です。

## [Slide 03]

> まず MLOps の工程全体を並べ、その後に各 GCP 技術の担当工程を見ます。

## [Slide 04]

> BigQuery、Dataform、Cloud Run、GCS、Pub/Sub が工程ごとに役割分担しています。

## [Slide 05]

> Dataform、embedding-job、training-job、search-api が一本の MLOps パイプラインでつながっています。

## [Slide 06]

> 特徴量定義は SQL と Python でずらさないことが重要です。

## [Slide 07]

> 物件埋め込みはバッチ、クエリ埋め込みはオンラインで処理します。

## [Slide 08]

> `/search` は意味検索と構造化条件検索を両方受けます。

## [Slide 09]

> `/search` は埋め込み、候補抽出、rerank、ログ保存の順で進みます。

## [Slide 10]

> BigQuery は保存先ではなく、VECTOR_SEARCH を担う検索基盤でもあります。

## [Slide 11]

> 最終順位は属性、行動、意味特徴をまとめて LightGBM が決めます。

## [Slide 12]

> booster 未配置でも `/search` は成立し、段階導入できます。

## [Slide 13]

> ranking_log と feedback_events が、次の学習材料になります。

## [Slide 14]

> 再学習は Scheduler と Pub/Sub と Eventarc でループ接続されています。

## [Slide 15]

> Cloud Run に API と Job を寄せることで、学習素材として説明しやすくなります。

## [Slide 16]

> デモではコードだけでなく BigQuery テーブルも合わせて見ます。

## [Slide 17]

> GCP 上の MLOps は、推論関数ではなく工程全体の設計です。
